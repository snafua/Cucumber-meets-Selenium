This allows developers to easyly build complex layouts without worrying how they will look on different devices .</P><P>There are only three tags you can use to create your layout: <CODE>b:container</CODE>, <CODE>b:row</CODE> and <CODE>b:column</CODE>.</P><P>The <CODE>b:container</CODE> is, as the name suggests, a container for <CODE>b:row</CODE>s and <CODE>b:column</CODE>s; it centers everything adding appropriate padding and margin.
        As a rule of thumb, always surround your rows with a container. Note that containers can't be nested. By default, containers make for a airy design, leaving a lot of the screen white, as you can see on the page you're currently reading
         (and similar to the layout in most books). 
        You can reduce the amount of white space by adding the <CODE>fluid</CODE> attribute: <CODE>&lt;b:container fluid="true"&gt;</CODE>. 
        </P><P><CODE>b:row</CODE> is a container for <CODE>b:column</CODE>s and you can't nest a <CODE>b:container</CODE> in a <CODE>b:column</CODE>.</P><P>BootsFaces takes full advantage of Bootstrap Grid system while maintaining the benefits of being a JSF framework.</P><P>This means that you can leverage Bootstrap's layout feature using for example the JSF templating system and conditional rendering to achieve and mantain very complex layouts without much effort.</P><P>This website is an example itself: it uses a page template with <CODE>ui:insert</CODE> and <CODE>ui:include</CODE> and the pages are <CODE>ui:composition</CODE>s.</P><H2>Understanding BootsFaces Grid System</H2><STRONG>Grid options</STRONG>
